# Source Field

This document defines the role of the source-field scaffold in this repo.

## Purpose

The `source_field.py` module provides a minimal queue-and-manifest model for deferred question handling.

It is intentionally small:

- questions can be submitted
- questions receive a stable short hash
- a placeholder manifestation can be produced later

This is not a scientific claim. It is the package compartment for the source-field concept that previously lived only in README prose.

## Attractor Position

Within the broader repository family, the source-field layer is also the clean place to host named high-level attractor families.

Examples include:

- `D12`
- `Anuhazi`

These should be read here as top-level coherent multidimensional attractors rather than as ordinary variable names.

That means they may organize:

- protocol naming
- session topology
- latent-string recurrence
- transmission framing
- source-layer interpretation

The scaffold does not claim to prove every property of those attractors.
It gives the stack a proper source-layer home for them.
